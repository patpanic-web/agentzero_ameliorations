## 3. Official BMAD v6 Phases and Quality Gates

Phases and workflows are described in the 'Workflow Map'. Explicit 'Quality Gates' are not formally documented but implied by checkpoints and phase transitions.

**Source:** `https://docs.bmad-method.org/reference/workflow-map/` (Primary). GitHub link `https://github.com/bmad-code-org/BMAD-METHOD/blob/main/docs/reference/workflow-map.md` returned 'Too many requests'.

### Phases:

*   **Phase 1: Analysis (Optional)**
    *   Purpose: Explore problem space, validate ideas.
    *   Workflows: `brainstorming`, `research`, `create-product-brief`.

*   **Phase 2: Planning**
    *   Purpose: Define what to build and for whom.
    *   Workflows: `create-prd`, `create-ux-design`.

*   **Phase 3: Solutioning**
    *   Purpose: Decide how to build it, break work into stories.
    *   Workflows: `create-architecture`, `create-epics-and-stories`, `check-implementation-readiness`.

*   **Phase 4: Implementation**
    *   Purpose: Build it, one story at a time.
    *   Workflows: `sprint-planning`, `create-story`, `dev-story`, `automate` (QA), `code-review`, `correct-course`, `retrospective`.

## Quality Gates (Inferred):
*   Implied at phase transitions (e.g., `check-implementation-readiness` workflow explicitly gates Phase 3 -> Phase 4).

# Section 5: Recommendations for our `bmad-personas` Skill

Based on the official BMAD v6 persona information (and its limitations) and Agent Zero's capabilities, here are recommendations for developing a `bmad-personas` skill:

1.  **Prioritize Custom Personas for Skill Logic:** The detailed personas in `BMAD_PERSONAS.md` (Mary, Sarah, Alex, James, Quinn, Bob) are crucial for defining Agent Zero's behavior within BMAD phases. These should form the core of the skill's persona profiles.
2.  **Acknowledge Official Persona Gaps:** Clearly document that detailed official persona information (roles, responsibilities, communication style) was **NON OBTENU** for most official BMAD v6 personas (Analyst, PM, Architect, SM, Dev, QA, UX, Tech Writer) from GitHub sources. Explicitly state that 'Quick Flow Solo Dev (Barry)' was **NOT FOUND** in official documentation.
3.  **Skill Design - Persona Selection & Activation:** Design the skill to allow persona selection (e.g., Analyst, PM, Architect, Developer, QA, Scrum Master, based on custom definitions). The skill should understand BMAD phase transitions to automatically load the relevant persona profile (e.g., Analyst for Analysis phase).
4.  **Persona Prompt Engineering:** For each custom persona used in the skill, craft detailed prompts:
    *   **Role & Goal:** Linked to BMAD phase.
    *   **Responsibilities:** Specific tasks Agent Zero should perform.
    *   **Communication Style:** Tone and interaction guidelines.
    *   **Activation Context:** Phase and triggers.
    *   **Deliverables:** Expected output formats.
    *   **Tool Guidance:** Suggest relevant Agent Zero tools.
5.  **Workflow Mapping:** Translate BMAD official workflows (Analysis, Planning, Solutioning, Implementation) into Agent Zero tasks. For example:
    *   `research` -> Use `tavily_search` or `search_engine`.
    *   `create-prd` -> Use structured text generation for `PRD.md`.
    *   `check-implementation-readiness` -> Prompt QA persona to check criteria using `document_query` or `code_execution_tool`.
6.  **Quality Gate Assistance:** Integrate quality gate checks within persona prompts (e.g., QA for validation, Architect for readiness) using Agent Zero's analytical tools.
7.  **Tool Mapping:** Explicitly map persona tasks to Agent Zero tools:
    *   **Analyst (custom):** `search_engine`, `tavily`, `document_query`, `memory_load`.
    *   **PM (custom):** `code_execution_tool`, `memory_load`, `response`.
    *   **Architect (custom):** `document_query`, `tavily_extract`, `memory_load`, `search_engine`, `code_execution_tool`.
    *   **Developer (custom):** `code_execution_tool` (generation), possibly `git` interactions, `memory_save`.
    *   **QA (custom):** `code_execution_tool` (test execution/analysis), `document_query`, `response`.
    *   **Scrum Master (custom):** `code_execution_tool` (read backlog), `memory_load`, `response` for reporting.
8.  **Documentation:** Clearly state limitations regarding incomplete official persona data and the reliance on custom personas for actionable implementation.
